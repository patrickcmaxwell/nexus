> so that **the system knows the exact requirements for that city**.
>

> - Checklist builder supports sections: Required Permits, Document List, Inspection Types, Fees, Special Forms
> - Each requirement item has fields: name, description, mandatory/optional, file type constraints, link to official source
> - Changes are versioned (v1 → v2) with effective date
> - Searchable by zip code → returns closest matching jurisdiction(s)
>

> As an **Admin**,  
> I want the system to **flag potentially outdated city rules**,  
> so that **we can keep the knowledge base accurate**.
>
