> ### NEW ALTITUDE WEBSITE PROCESS – v0 Edition (Feb 2026)
> 
> **Phase 0 – Intake (Nick – 100% his job, no exceptions)**
> - Nick runs the intro call (or skips it if the client is low-touch).
> - Nick creates the Click-Up task **and forces the client to fill a standardized intake form** (we’ll make this in Click-Up or a simple Tally link).
> - Required fields (Nick cannot mark the task “Ready for Build” until every single one is complete):
>   - Dropbox folder with all assets + logos
>   - Domain registrar login or current host access
>   - Previous website link + 3 things they love + 3 things they hate about it
>   - 2–3 competitor/reference sites they want to feel like
>   - Exact color hex codes + brand fonts (or “match this screenshot”)
>   - Full page list (Home, About, Services, Contact, etc.)
>   - Who is providing copy? (client / us / copywriter)
>   - Who is providing photos? 
>   - Must-have functionality (forms, booking calendar, shop, blog, members area, etc.)
>   - Target audience + 1-sentence goal of the site
>   - Any “non-negotiables”
> 
> Nick screenshots the reference sites + colors and pastes everything into one clean Click-Up comment.  
> If anything is missing → Nick chases it. Patrick does **not** start until the task is tagged “Intake Complete”.
> 
> **Phase 1 – First Draft (Patrick – 30–60 min)**
> - Patrick opens v0, pastes the entire Click-Up intake as the prompt + uploads the reference screenshots.
> - Generates full multi-page site (or at minimum Homepage + 2 key pages).
> - Deploys instant Vercel preview link.
> - Drops the link + “Ready for client review” comment in Click-Up.
> - That’s it. Patrick’s done for now.
> 
> **Phase 2 – Client Review & Iterations (Nick runs point, Patrick only touches v0)**
> - Nick emails the preview link to the client: “Here’s your AI-generated first draft based on everything you told us. Reply with bullet-point feedback.”
> - All client feedback goes **only** to Nick.
> - Nick consolidates feedback into 1–3 clean bullet points in Click-Up (no novel-length emails).
> - Patrick jumps back into the **same v0 thread**, types the 3 bullets, regenerates → new preview link in <15 min.
> - Repeat max 2–3 rounds (we will set that expectation with the client on intake).
> 
> **Phase 3 – Polish & SEO (Patrick – 20–30 min)**
> - Once client says “looks good”, Patrick:
>   - Adds basic on-page SEO (titles, metas, headings, alt text, Open Graph)
>   - Quick mobile check + speed tweaks
>   - Connects any forms / analytics
> - Pings Nick: “Final version ready – green light to launch?”
> 
> **Phase 4 – Launch (Nick owns)**
> - Nick double-checks on mobile + desktop.
> - Nick gives Patrick domain/host access if needed.
> - Patrick pushes to production (one click on Vercel).
> - Nick immediately sends the “YOUR NEW WEBSITE IS LIVE!” email + onboarding sequence.
> 
> ### Why this is dramatically less work for both of us
> 
> - Patrick touches each site 3–4 times max instead of 8–10.
> - No more “what does the client actually want?” guessing — the intake forces clarity before any code is touched.
> - v0 does 85% of the design/dev work. We only do light custom code on the last pass if truly needed.
> - Nick stays the single point of client contact (exactly what he wants), but now he has a crystal-clear spec to work from so he doesn’t accidentally misrepresent the client’s needs.
> - Total build time per site drops from 8–15 hours to 2–4 hours.
