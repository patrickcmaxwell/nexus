> # Magic Bag App: Product Requirements Document (PRD)
> 
> ## 1. Overview
> ### App Name
> Magic Bag
> 
> ### Description
> Magic Bag is a cross-platform mobile application designed to assist users with ADHD (and similar neurodiverse conditions) in starting their day gently and effectively. The app provides personalized, low-pressure micro-tasks upon waking, adapts to the user's "brain mode" (e.g., focus, hyper, or crash states), and includes an optional social component for anonymous, supportive sharing of daily experiences. The goal is to reduce morning overwhelm by offering simple, actionable suggestions that build momentum without gamification or judgment.
> 
> ### Target Audience
> - Individuals with ADHD/ADD or those who struggle with morning routines due to focus, energy, or motivation issues.
> - Ages 18+ (assuming adult users, but adaptable for teens with parental consent).
> - Users seeking a non-intrusive, adaptive tool rather than a rigid productivity app.
> 
> ### Platforms
> - Cross-platform: iOS (minimum version 14.0) and Android (minimum version 8.0).
> - Built using frameworks like Flutter or React Native for shared codebase efficiency.
> 
> ### Key Principles
> - **Gentle and Adaptive**: No streaks, badges, or penalties—focus on positive reinforcement and personalization.
> - **Privacy-First**: All data (ratings, modes) stored locally or anonymized if shared socially.
> - **Accessibility**: Voice interactions, large text, color-blind friendly, and ADHD-friendly UI (minimal distractions).
> - **Offline-First**: Core features (tasks, ratings) work without internet; social features require connectivity.
> 
> ## 2. Goals and Objectives
> - **Primary Goal**: Help users build morning momentum through micro-tasks that align with their current mental state, reducing decision paralysis.
> - **Secondary Goals**:
>   - Adapt suggestions over time based on user feedback to improve relevance.
>   - Foster a sense of community without overwhelming social pressure via anonymous sharing.
>   - Achieve high user retention through simplicity and positive experiences (target: 70% weekly active users after 30 days).
> - **Success Metrics**:
>   - Daily task completion rate > 50%.
>   - User satisfaction score (via in-app feedback) > 4/5.
>   - App Store ratings > 4.5 stars.
> 
> ## 3. Key Features
> ### 3.1 Core Features
> - **Adaptive Wake-Up System**:
>   - Customizable alarm with gentle sounds (e.g., birds, waves) or voice nudge (e.g., "Good morning, ready for a gentle start?").
>   - User sets mood upon waking (via quick tap: Groggy, Wired, Neutral) to influence task selection.
>   - Fades in volume to avoid startling users.
> 
> - **Micro-Tasks Generation**:
>   - Delivers 3-5 simple tasks per morning session (30 seconds to 5 minutes each).
>   - Examples: "Sip a glass of water while looking out the window," "Stretch your arms for 30 seconds," "Text a friend 'good morning'."
>   - Tasks drawn from a library of 50+ pre-defined options, categorized by energy level.
>   - One-tap completion (check-off) with optional voice command support.
> 
> - **Task Rating and Learning Engine**:
>   - After each task or session, users rate via voice/text: +1 (felt good), 0 (neutral), -1 (not helpful).
>   - App uses simple machine learning (local ML model, e.g., via TensorFlow Lite) to adapt future suggestions—prioritizes high-rated tasks, avoids low-rated ones.
>   - Maintains a personal "diary" view showing patterns (e.g., "You enjoy stretching on hyper days").
> 
> - **Brain Modes**:
>   - Toggleable modes to customize tasks:
>     - **Focus Mode**: Tiny, low-effort tasks for scattered brains (e.g., "Breathe deeply 5 times").
>     - **Hyper Mode**: Energy-burning tasks (e.g., "Dance while making coffee").
>     - **Crash Mode**: Restful cues (e.g., "Sit with a weighted blanket for 2 minutes").
>   - Auto-detect option based on past patterns or quick morning quiz.
> 
> ### 3.2 Social Features
> - **Quiet Social Feed**:
>   - Optional tab for viewing anonymous "sparks" from other users' mornings (e.g., "Someone breathed with the sunrise," "Stood on one foot for 30 seconds").
>   - No usernames, profiles, or direct messaging—fully anonymized.
>   - Users can share their completed tasks as "sparks" with one tap (e.g., "Shared: Sipped water by the window").
> 
> - **Light Interactions**:
>   - Tap to send soft reactions: "Me too" or "Nice" (counts as anonymous encouragement).
>   - Feed filtered by global trends or similar brain modes (e.g., see other "hyper mode" sparks).
>   - Privacy controls: Opt-in only; no data sold or tracked beyond app usage.
> 
> - **Community Guidelines Integration**:
>   - Built-in prompts to encourage positive, supportive sharing.
>   - Moderation: AI-flagged for any non-anonymous or harmful content (rare due to design).
> 
> ### 3.3 Additional Features
> - **Task Library Customization**:
>   - Users can add/pin personal tasks (e.g., "Water the plant").
>   - Searchable library with filters (e.g., by time, energy).
> 
> - **Dashboard and History**:
>   - Clean home screen: Today's tasks at top, brain mode selector, social tab button.
>   - Weekly summary: "You completed 15 tasks—stretching was your favorite!"
> 
> - **Notifications**:
>   - Gentle reminders (e.g., "Ready for tomorrow's suggestions? Rate today's.").
>   - Opt-out for all non-essential pushes.
> 
> - **Accessibility and Settings**:
>   - Voice-over support, dark mode, font size adjustments.
>   - Data export/delete options for privacy compliance (GDPR/CCPA).
> 
> ## 4. User Flows
> ### 4.1 Onboarding Flow
> 1. Download and launch app.
> 2. Quick setup: Set alarm time, select default brain mode, grant permissions (mic for voice, notifications).
> 3. Tutorial: Simulated morning session with 2 sample tasks.
> 4. Opt-in to social features with privacy explanation.
> 
> ### 4.2 Morning Routine Flow
> 1. Alarm triggers: Gentle sound/voice.
> 2. Open app: Displays 3-5 tasks based on mode.
> 3. Complete tasks: One-tap check, optional skip.
> 4. Rate session: Voice/text input.
> 5. Optional: Share to social feed or view others' sparks.
> 6. End: "Great start! See you tomorrow."
> 
> ### 4.3 Social Interaction Flow
> 1. Tap social tab.
> 2. View feed of anonymous sparks.
> 3. Tap a spark: Send "Me too" or "Nice."
> 4. Share own spark: Select from completed tasks, anonymize, post.
> 
> ## 5. Technical Requirements
> ### 5.1 Architecture
> - **Frontend**: Flutter/React Native for UI consistency across platforms.
> - **Backend**: Minimal—use Firebase or Supabase for social features (anonymous data syncing); all personalization local.
> - **Database**: Local storage (SQLite) for tasks/ratings; cloud for social (anonymized).
> 
> ### 5.2 Integrations
> - Voice API: Integrate with device speech-to-text (e.g., Google Speech on Android, SiriKit on iOS).
> - ML Model: Lightweight on-device model for task adaptation (no cloud dependency).
> - Analytics: Anonymous usage tracking (e.g., via Mixpanel) for improvements.
> 
> ### 5.3 Security and Compliance
> - Data Encryption: All local data encrypted.
> - Anonymity: Social posts stripped of identifiers.
> - Compliance: ADA accessibility, COPPA (if under 13, but target 18+), app store guidelines.
> 
> ### 5.4 Performance
> - Offline Mode: Tasks and ratings work without internet.
> - Battery Optimization: Minimal background activity.
> - App Size: Target < 50MB.
> 
> ## 6. Non-Functional Requirements
> - **UI/UX Guidelines**:
>   - Minimalist design: Pastel colors, large buttons, no clutter.
>   - Use tables for summaries (e.g., weekly history as a simple table).

>   - Unit tests for task generation and ratings.
>   - User beta testing with ADHD focus groups.
> - **Monetization** (Optional for MVP):
>   - Free core app; premium for advanced customizations (e.g., unlimited task library).

> - **MVP (Version 1.0)**: Core wake-up, tasks, ratings, brain modes.
